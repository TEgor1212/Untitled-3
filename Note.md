## Конспект по Bash

### Основы

- Bash — интерпретатор команд в Unix-подобных системах.
- Скрипты сохраняются в файлах `.sh`, запускаются `bash script.sh`.
- Команды раздельные: `;`, `&&` (выполнить следующую при успехе), `||` (при ошибке).
- Переменные создаются без пробелов: `name="Егор"`. Доступ: `$name` или `${name}`.
- Встроенные переменные: `$USER`, `$HOME`, `$PWD`, `$?` (код последней команды).

### Работа с файлами и каталогами

- `pwd` — показать текущую директорию.
- `ls -la` — показать файлы со скрытыми и подробной информацией.
- `cd path` — перейти в каталог; `cd -` вернуться; `cd ~` в домашний.
- `cp src dst`, `mv src dst`, `rm file`, `rm -r dir`, `mkdir -p dir/subdir`.
- Потоки: `>` перезапись, `>>` дописывание, `<` ввод из файла; `|` передача вывода на ввод.

### Переменные окружения и экспорт

- `export VAR=value` — сделать переменную доступной дочерним процессам.
- Просмотр окружения: `env`, `printenv`, `set`.
- Удаление: `unset VAR`.

### Управление процессами

- `ps`, `ps aux`, `top`, `htop` — список процессов.
- `&` — запуск в фоне; `jobs` — список фоновых задач.
- `fg %1`, `bg %1` — вернуть/продолжить задачу.
- `kill PID`, `kill -9 PID` — завершение процесса.

### Циклы и условия

```bash
if [ -f "$file" ]; then
  echo "Есть файл"
fi

for item in *.txt; do
  echo "$item"
done
```

- Условные выражения: `[ ]` или `[[ ]]`; проверки `-f`, `-d`, `-z`, `-n`.
- `case "$var" in pattern) ... ;; *) ... ;; esac`

### Функции

```bash
greet() {
  echo "Привет, $1"
}
greet "мир"
```

- Параметры доступны как `$1`, `$2`, `$@`; `"$@"` сохраняет слова.

### Отладка и лучшие практики

- `set -e` — остановка при ошибке; `set -u` — ошибка при неустановленных переменных.
- `set -o pipefail` — передача кода ошибки из пайплайна.
- `bash -x script.sh` — трассировка выполнения.
- Всегда цитируйте переменные `"${var}"`, используйте функции для повторно используемых шагов, документируйте.
